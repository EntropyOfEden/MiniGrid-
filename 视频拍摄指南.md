# 视频拍摄指南

本文档说明如何在本项目中**录制大作业讲解视频**：你负责讲解与拍摄，已提供检查点（如 `runs/.../agent.pt`）与指标（`metrics.jsonl`）。按下面顺序准备，可减少补拍。

---

## 1. 拍摄前准备

### 1.1 环境

- 安装 Python 3.10+，在项目根目录执行：`pip install -r requirements.txt`
- 确认能弹出游戏窗口：`python evaluate.py --ckpt <路径>/agent.pt --episodes 5 --render`（这是使用模型运行5次游戏）  
  - `<路径>` 用训练同学给的 **`agent.pt` 完整路径**（须与验收环境一致）。
- **麦克风**：环境安静；先试录 10 秒听底噪与音量。
- **画面**：建议 1920×1080 或课程要求分辨率；字体放大便于看清代码与终端。

### 1.2 需要确认的信息

- 默认或实际使用的 **`--env-id`**（如 `MiniGrid-DoorKey-8x8-v0`）
- 用于演示的 **`agent.pt` 路径**（是否与 `runs/README.md` 里某目录对应）
- 是否有 **`runs/compare_dqn_double.png`** 或两条 `metrics.jsonl` 可对比 Vanilla / Double DQN

---

## 2. 推荐录制结构（可按课程时长裁剪）

| 段落 | 内容 | 建议时长 |
|------|------|----------|
| 片头 | 课题名称、我们两个的姓名、**分工一句**（谁训练 / 谁讲解） | 约 20–40 秒 |
| 任务 | MiniGrid 画面：`--render` 跑 1 回合；口头说明「钥匙 → 门 → 绿格」 | 约 1 分钟 |
| 算法 | DQN：回放 + 目标网络 + \(\epsilon\) 贪婪；Double DQN：一句话说清「在线选动作、目标网估值」 | 约 1–2 分钟 |
| 训练结果 | 打开 `metrics.jsonl`、或展示对比图 / 两条曲线；说明「回报或成功率随时间上升」 | 约 1 分钟 |
| 代码 | IDE 中 brief 展示 `dqn_agent.py`（Double 分支）、`networks.py`、`train.py` 中与探索/目标网络相关处 | 约 1–2 分钟 |
| 演示 | 再录 **`evaluate.py --render` 成功完成任务** 的 1–2 个完整回合（可加速剪辑） | 约 1–2 分钟 |
| 结尾 | 改进点小结、致谢 | 约 20 秒 |

总时长按课程要求调整；**算法 + 演示**不要缺。

---

## 3. 录屏与窗口捕获（Windows）

- **系统自带**：Win+G 打开 Xbox Game Bar，可录屏（注意是否录到麦克风）。
- **OBS Studio**：添加「窗口捕获」选 MiniGrid / Pygame 窗口；另开显示器捕获或显示器区域用于 IDE。
- **建议**：终端里先 `cd` 到项目根，把要念的命令复制好再开录，避免现场打错路径。
- **演示命令示例**（路径换成你的 `agent.pt`）：

```text
python evaluate.py --ckpt runs/double_dqn-DoorKey-8x8-toward-6/agent.pt --episodes 5 --render
```

若检查点在其他目录，只改 `--ckpt` 即可；**一般不要擅自改小 `--max-steps`**，以免任务未完成却被截断。

---

## 4. 讲解时注意点

- **与仓库一致**：口头说的环境名、算法名（DQN / Double DQN）须与 `作品说明.md`、实际训练配置一致。
- **`metrics.jsonl`**：可说明列里 `reward`、`task_success` 或回报大致趋势；不必逐行读。
- **对比实验**：若有两份模型，可说「基线是 Vanilla，改进是 Double DQN，后者曲线更稳或完成率更高」——以**你手头的数据为准**，勿夸大。
- **评估脚本**：若演示里偶有「非最优」动作，可说明评估脚本里存在少量随机性用于防卡死；**重点展示能成功通关的片段**即可。
